# Changelog

## 1.3.0 — 2026-06-23

### Исправлено / улучшено
- **Расшифровка перенесена в фоновый процесс расширения.** Раньше `qpdf` (WASM)
  запускался в странице и блокировался её CSP — поэтому учебник скачивался, но
  оставался под паролем. Теперь страница только скачивает файл и отдаёт байты в
  background, где `qpdf` снимает пароль (контекст расширения не подчиняется CSP
  страницы). Добавлен `content_security_policy: wasm-unsafe-eval`.
- **Запоминание пароля последнего учебника.** Пароль сохраняется в
  `storage.local`: даже если закрыть и снова открыть расширение, поле «Пароль от
  документа» покажет пароль последнего скачанного учебника (с именем файла).
  Кнопка «✕» — забыть пароль.

### Технически
- Байты PDF передаются между страницей и background в base64 (компактно и
  безопасно для messaging). Пайплайн «base64 → qpdf --decrypt → base64» проверен.
- `qpdf.js` грузится в background: через `importScripts` (Chrome/Yandex SW) и
  через `background.scripts` (Firefox). `web_accessible_resources` больше не нужны.

## 1.2.1 — 2026-06-23

### Улучшено
- Если автоматически снять пароль не удалось, расширение **показывает пароль
  отдельным полем** в попапе («Пароль от документа») и копирует его в буфер —
  остаётся вставить в окно «Необходимо ввести пароль».
- Надёжнее поиск пароля: сканируются и `localStorage`, и `sessionStorage`, с
  более гибким разбором ключа `ispring::book/<пароль>`.
- Понятные сообщения для трёх случаев: разблокировано / пароль ниже / пароль не
  найден (откройте учебник один раз и повторите).

## 1.2.0 — 2026-06-23

### Добавлено
- **Автоматическое снятие пароля с PDF.** В расширение встроен `qpdf`,
  скомпилированный в WebAssembly (`@jspawn/qpdf-wasm`, Apache-2.0). Теперь файл
  скачивается, расшифровывается прямо в браузере найденным паролем и
  сохраняется **уже разблокированным** — пароль вводить не нужно, документ можно
  свободно пересылать.
- Загрузка и расшифровка идут **во вкладке** Росдистанта (своя сессия и
  localStorage), результат сохраняется через Blob.

### Изменено
- Если расшифровать не удалось (нет пароля, другой тип защиты, ограничения
  страницы) — расширение, как и раньше, скачивает исходный файл и показывает
  пароль. Регрессии нет.
- `web_accessible_resources` открывает `lib/qpdf.wasm` для доменов Росдистанта.

## 1.1.0 — 2026-06-23

### Добавлено
- **Автоматическая выдача пароля от PDF.** Учебники Росдистанта — это PDF,
  защищённые паролем. Расширение достаёт пароль из `localStorage`
  (`ispring::book/<пароль>`), показывает его в попапе и копирует в буфер обмена,
  чтобы сразу вставить в окно «Необходимо ввести пароль».

## 1.0.0 — 2026-06-23

Первый релиз «Помощь росдистантикам / Rosdistant Helper».

### Добавлено
- **Скачивание учебников в один клик** с платформы Росдистант:
  - по вставленной ссылке (расширение само открывает её в активной сессии);
  - с текущей открытой страницы учебника.
- Автоматическое извлечение `fileOpenParams.filePath` из просмотрщика iSpring и
  загрузка исходного файла через `downloads` API с сессионными cookie.
- Запасной режим: сборка изображений страниц в печатный PDF.
- Перенос экспорта тестов из `opiums9/rosdistant_export_test`:
  сохранение теста в HTML и в PDF.
- Новый интерфейс попапа на русском языке.

### Совместимость
- Chrome, Yandex, Edge, Opera (MV3, `service_worker`).
- Firefox (MV3, `background.scripts` + `browser_specific_settings`).
- Safari (через официальный конвертер Apple, сборка на macOS).

### Сборка
- Zero-dependency скрипты `scripts/build.mjs` и `scripts/pack.mjs` на Node.js.
