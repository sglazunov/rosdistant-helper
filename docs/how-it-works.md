# Как это работает (технически)

## Платформа

Росдистант (`edu.rosdistant.ru`) — это **Moodle** с темой Lambda. Учебники
встроены как контент **iSpring** (HTML5-просмотрщик `data/js/viewer.js`).
Доступ к материалам — только после авторизации (сессионные cookie Moodle).

## Учебники

На странице открытого учебника iSpring оставляет в разметке объект:

```js
var fileOpenParams = { "filePath": "…", "title": "…", … };
```

`filePath` — ссылка (обычно относительная) на **исходный файл** учебника.

Алгоритм кнопки **«Скачать учебник»** (`src/background.js`):

1. `inj_extractBook()` внедряется во все фреймы страницы и ищет
   `var fileOpenParams = {…}` в HTML, затем разбирает его как JSON.
2. `filePath` превращается в абсолютный URL относительно адреса страницы.
3. URL передаётся в `chrome.downloads.download(...)`. Загрузка идёт **с cookie
   текущей сессии**, поэтому файл за авторизацией скачивается штатно.
4. Имя файла берётся из `title` и очищается от недопустимых символов;
   расширение файла — из URL.

### Режим «по ссылке»

Если в попап вставлена ссылка, фоновый воркер:

1. проверяет, что домен — `*.rosdistant.ru`;
2. открывает ссылку в **фоновой вкладке** (в вашей же сессии);
3. опрашивает страницу несколько раз, пока iSpring не вставит `fileOpenParams`
   (`extractWithRetries`);
4. скачивает файл и закрывает вкладку.

### Пароль от PDF

Учебники Росдистанта отдаются как **PDF, защищённые паролем**. Сам пароль
iSpring хранит в `localStorage` страницы в ключе вида `ispring::book/<ПАРОЛЬ>`.
`inj_extractBook()` перебирает `localStorage`, берёт запись с самым свежим
`updated` и возвращает пароль. Попап показывает его в отдельной плашке и
**автоматически копирует в буфер обмена** — остаётся вставить его в окно
«Необходимо ввести пароль». (В исходном проекте это была строка
`confirm(passw.key)`.)

### Снятие пароля (расшифровка)

Учебники Росдистанта — это PDF, которые iSpring **намеренно пишет не по
стандарту**: у потоков с картинками стоит `/Length 0` (плюс фейковый ключ `/F`).
Строгие инструменты (`qpdf`) на этом падают с `expected endstream`. Поэтому
используется **pdf.js** — тот же движок, которым браузеры эти учебники и
показывают.

1. `lib/pdf.min.js` + `lib/jspdf.umd.min.js` + `lib/rd-decrypt.js` внедряются во
   все фреймы страницы. `web_accessible_resources` открывает
   `lib/pdf.worker.min.js` для доменов Росдистанта.
2. `rd-decrypt.js` в нужном фрейме: читает `fileOpenParams` и пароль из
   `localStorage`, скачивает файл через `fetch(..., {credentials:"include"})`.
3. Воркер pdf.js поднимается из **same-origin blob** (чтобы страница могла его
   запустить), документ открывается с паролем.
4. Каждая страница рендерится в `<canvas>`, картинка кладётся в новый PDF через
   **jsPDF**. Прогресс шлётся в background → проценты на иконке расширения.
5. Готовый PDF (валидный `%PDF`, без `/Encrypt`) сохраняется через `Blob` —
   открывается без пароля.

Итог — постраничный (изображения), без текстового слоя, зато гарантированно
открывается и пересылается. Если расшифровать не удалось — скачивается исходный
файл, а пароль показывается и запоминается в попапе. Регрессии нет.

Библиотеки: [pdf.js](https://github.com/mozilla/pdf.js) (Apache-2.0),
[jsPDF](https://github.com/parallax/jsPDF) (MIT).

### Запасной вариант

Если прямого файла нет, `inj_extractBook()` собирает изображения страниц
(`/res/`, `/data/`, `/pages/`, `/slides/`), а `inj_printImages()` верстает из
них печатный документ и открывает диалог печати → **«Сохранить как PDF»**.

## Тесты

Перенесено из исходного проекта `opiums9/rosdistant_export_test`:

- `inj_exportTestHtml()` — сохраняет `form.questionflagsaveform` + хлебные
  крошки как HTML со стилями темы.
- `inj_exportTestPrint()` — то же во временном iframe + `print()` для PDF.

## Права в манифесте

| Право | Зачем |
| --- | --- |
| `scripting` | внедрять функции извлечения в страницу |
| `activeTab` / `tabs` | работать с текущей вкладкой и открывать ссылку |
| `downloads` | сохранять файл учебника с сессионными cookie |
| `storage` | задел под настройки |
| `host_permissions: *.rosdistant.ru` | доступ только к домену Росдистанта |

Расширение **не** запрашивает доступ к другим сайтам и не отправляет данные
куда-либо — вся работа идёт локально в браузере.
