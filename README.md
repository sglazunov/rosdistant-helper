# Помощь росдистантикам · Rosdistant Helper

Браузерное расширение, которое **скачивает учебники** и **сохраняет тесты** с
платформы [Росдистант](https://rosdistant.ru) (`edu.rosdistant.ru`, Moodle +
iSpring) **в один клик** — по ссылке или прямо с открытой страницы.

Это развитие проекта [`opiums9/rosdistant_export_test`](https://github.com/opiums9/rosdistant_export_test):
к экспорту тестов добавлена **автоматическая загрузка учебников** и поддержка
**Chrome, Yandex, Edge, Opera, Firefox и Safari**.

> ⚠️ Учебники на Росдистанте доступны только после авторизации. Расширение
> ничего не взламывает — оно использует **вашу собственную активную сессию** в
> том же браузере. Скачивайте только материалы, к которым у вас уже есть доступ.

---

## Что умеет

| Кнопка | Что делает |
| --- | --- |
| **⬇️ Скачать учебник** | Находит просмотрщик учебника (iSpring), достаёт исходный PDF, **снимает с него пароль встроенным qpdf (WebAssembly)** и сохраняет уже разблокированный файл — его можно открыть без пароля и переслать. Можно **вставить ссылку** — расширение само откроет её в вашей сессии, дождётся загрузки и скачает учебник. |
| **💾 Сохранить тест в HTML** | Сохраняет завершённую попытку теста как HTML со стилями. |
| **🖨️ Сохранить тест в PDF** | Открывает диалог печати для сохранения теста в PDF. |

Если прямой файл учебника недоступен, расширение собирает изображения страниц и
открывает диалог печати → **«Сохранить как PDF»**. Если пароль снять не удалось —
файл скачивается как есть, а пароль показывается в попапе (с копированием в буфер).

## Как это работает

Учебники на Росдистанте — это контент iSpring, встроенный в страницу Moodle.
На странице учебника присутствует `var fileOpenParams = { … }` со ссылкой на
исходный файл (`filePath`). Расширение:

1. внедряет небольшой скрипт в страницу Росдистанта (только на доменах `*.rosdistant.ru`);
2. читает `fileOpenParams` и превращает `filePath` в абсолютную ссылку;
3. скачивает файл через `downloads` API — запрос автоматически уходит **с вашими
   cookie**, поэтому файлы за авторизацией скачиваются как обычно.

Подробнее: [`docs/how-it-works.md`](docs/how-it-works.md).

## Установка

Готовые папки для загрузки лежат в `dist/` (или соберите сами — см. ниже).

- **Chrome / Yandex / Edge / Opera** → [`docs/install-chrome.md`](docs/install-chrome.md)
- **Firefox** → [`docs/install-firefox.md`](docs/install-firefox.md)
- **Safari** (нужен Mac + Xcode) → [`docs/install-safari.md`](docs/install-safari.md)

Коротко (Chromium): `chrome://extensions` → включить **Режим разработчика** →
**Загрузить распакованное расширение** → выбрать папку `dist/chrome`.

## Сборка из исходников

Нужен [Node.js](https://nodejs.org) 18+. Зависимостей нет.

```bash
npm run build      # собирает dist/chrome и dist/firefox
npm run zip        # то же + .zip-архивы в dist/
```

Исходники — в `src/` (общие для всех браузеров). Билд просто копирует их в
`dist/<браузер>` и подставляет нужный `manifest.json`. После правок в `src/`
запустите `npm run build` заново.

Структура:

```
src/                 общий исходный код
  manifest.chrome.json   манифест для Chromium
  manifest.firefox.json  манифест для Firefox
  background.js          вся логика: извлечение и скачивание
  popup.html/.css/.js    интерфейс
  icons/
scripts/
  build.mjs          сборка dist/chrome и dist/firefox
  pack.mjs           упаковка в .zip
  build-safari.sh    конвертация в проект Safari (только macOS)
docs/                инструкции по установке
```

## Поддержка браузеров

| Браузер | Статус | Манифест |
| --- | --- | --- |
| Chrome, Yandex, Edge, Opera | ✅ готовая папка `dist/chrome` | MV3 `service_worker` |
| Firefox | ✅ готовая папка `dist/firefox` | MV3 `background.scripts` + `browser_specific_settings` |
| Safari | ✅ через конвертер Apple (`scripts/build-safari.sh`, нужен Mac) | тот же MV3 |
